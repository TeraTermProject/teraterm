Bug fixes: Tera Term Pro 2.3

  1. Tera Term window size could not be changed after the window size is smaller than 84 dot(94 dot on XP LUNA style). This is Tera Term Pro 2.3 original bug.
  2. A tab at end-of-line would lost.
  3. BOF(Buffer Over Flow) problem in ttpmacro.exe program, which 256 byte longer command line parameter is specified.
  4. BOF(Buffer Over Flow) problem in ttpmacro.exe program when a line of macro is longer than 256 byte.
  5. The program of ttermpro.exe can not send the server data to ttpmacro.exe program with DDE transaction.
  6. The 'pause' macro command had a time-lag about 1 sec.
  7. The macro script might stall with logging mode.
  8. The cpu scheduling of Tera Term process could occupy while macro script running.
  9. The division operator of macro language does not work.
  10. Tera Term would crash when the sending file size exceeded 20MB with ZMODEM protocol.
