Additional settings / "Serial port" tab ([Setup] menu)

Settings related to the serial port.

Port:
Sets the default serial port or displays the currently connected serial port.

When the serial port is not connected, Sets the default serial port for New connection Dialog.
A non existent serial port can be set. New connection dialog does not show any non existent serial ports.
By default, maximum serial port number in the list is 256. It is configurable in MaxComPort setting.
You cannot start connecting from this dialog. (version 5.4.0 or later) You should use New connection dialog for connecting to a serial port.

When the serial port is connected, displays the serial port to which it is connected.
You cannot change the port number while connected from this dialog. (version 5.4.0 or later) To change the connection port number, disconnect once using [File]-[Disconnect] menu and connect to other port using the New connection dialog.

Serial parameters
Serial port parameters. Choose the appropriate parameters for the modem or the host.
Whether the selected or specified settings are actually applied depends on serial port chip or driver.

When the serial port is not connected, Sets the parameters for connecting to the serial port.

When the serial port is connected, Set parameters for the connected serial port.

Speed:
The Speed can be specified with any number.
Data:
8 bit / 7 bit

Note: If Data is 7 bit, the XMODEM, ZMODEM, B PLUS and Quick-VAN protocols are disabled.

Parity:
none / odd / even / mark / space
Stop bits:
1 bit / 2 bit
Flow control:
Xon/Xoff
Software flow

Note: If Flow control is Xon/Xoff, the XMODEM and Quick-VAN protocols are disabled.

RTS/CTS
Hardware flow(RTS/CTS)

Note: The "hardware" prior to version 4.104 applies to RTS/CTS.

DSR/DTR
Hardware flow(DSR/DTR)
none
No flow control
CTS, DSR, RING, RLSD
Displays the status of the control signals.
  • CTS (Clear To Send) : Ready to send
  • DSR (Data Set Ready) : Data set ready
  • RING (Ring Indicator) : Incoming (ringing)
  • RLSD (Receive Line Signal Detect) : Carrier detected
RTS
Displays the status of the RTS (Request To Send) control signal.
Set the RTS flow control method to Disable / Enable / Handshake / Toggle. For details, see the description of the fRtsControl member of the DCB structure.
  • Disable : always low
  • Enable : always high
  • Handshake : high when ready to receive
  • Toggle : high while transmitting
When RTS flow control is set to Disable or Enable, clicking RTS temporarily toggles the signal between low and high.
DTR
Displays the status of the DTR (Data Terminal Ready) control signal.
Set the DTR flow control method to Disable / Enable / Handshake. For details, see the description of the fDtrControl member of the DCB structure.
  • Disable : always low
  • Enable : always high
  • Handshake : high when ready to receive
When DTR flow control is set to Disable or Enable, clicking DTR temporarily toggles the signal between low and high.
Transmit delay
Time intervals between characters (lines) being sent, in milliseconds. The unit is milliseconds. You can set the wait time per character (line). If the modem or host drops characters when Tera Term sends a large number of characters, try setting an appropriate delay time.
Detail information of COM port
The detail information of selected port is shown in the text box.
You can select all data by using CTRL+A. You can scroll horizontally by using CTRL/SHIFT + wheel.
If the transmitted or received data is dropped
Data sent or received may be lost for various reasons.
See the Dropping sent and received data section for information on the locations of drops and recommended countermeasures.
