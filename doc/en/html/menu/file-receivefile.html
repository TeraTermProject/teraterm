Receive file ([File] menu)

Receive a file from the host.

Selecting this menu brings up the "Receive file" dialog.
If you select this menu while holding down the CTRL key, it skips the "Receive File" dialog, allowing you to receive a file by simply selecting it.
At this time, the same receiving options as before will be used.

This behavior can be changed with Special Option.

"Receive file" dialog box (to input the file name)

Filename (drop file in this dialog)
Enter or select a filename to receive.
By dropping a file into dialog, You can also enter file name.
Binary
It will receive the contents of the file as is.
Text mode (character encoding, newline character conversion, suppression of control character output) is not implemented, so it is fixed to binary mode.
Auto-stop wait time(sec)
If there is no received data for the specified duration, the file reception will be terminated.
The default value for the wait time is 5 seconds.
If 0 seconds is specified, the file reception will not be terminated and will continue to wait for data.

Special options in the Tera Term setup file

  • Skip Receive file dialog

Drop of received data

When receiving data at a high bit rate from the serial port, there may be instances where received data is dropped.
This is likely to occur when the PC is under high load and Tera Term's processing (such as reading data from the UART chip, logging/file output, screen rendering, etc.) cannot keep up. Please also refer to "Send file" ([File] menu) Send size

"Receive file" dialog box (displayed while sending)

Cancel
Pressing the Cancel button will stop the file reception.
